# Research Sources Reference

Static reference for the design-log skill's research phase. Loaded on demand, not on every invocation.

## Key Books by Domain

| Domain | Key Books |
|--------|-----------|
| Error Handling / Resilience | "Release It!" — Michael Nygaard |
| UX / Usability | "Don't Make Me Think" — Steve Krug |
| API Design | "Designing Web APIs" — Brenda Jin |
| Security | "The Web Application Hacker's Handbook" |
| Performance | "High Performance Browser Networking" — Ilya Grigorik |
| CSS / Layout | "Every Layout" — Heydon Pickering & Andy Bell |
| JavaScript Patterns | "JavaScript Patterns" — Stoyan Stefanov |
| System Design | "Designing Data-Intensive Applications" — Martin Kleppmann |
| Accessibility | "Inclusive Design Patterns" — Heydon Pickering |
| Forms / Validation | "Form Design Patterns" — Adam Silver |
| State Management | "State Machines in JavaScript" — David Khourshid |
| Testing | "Testing JavaScript Applications" — Lucas da Costa |
| Email | "Email Marketing Rules" — Chad S. White |
| Automation | "Workflow" — Geoff Burch; workflow-tool official docs |

This is NOT exhaustive — use judgment to find the most relevant books for the specific task.

- How do top-tier products (Stripe, Linear, Notion, GitHub, Vercel) solve this specific problem?

- Postmortems if the task involves fixing reliability issues
